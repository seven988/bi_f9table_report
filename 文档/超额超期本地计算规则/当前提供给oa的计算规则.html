一、超额（Overdraft） 统一规则

1. 计算范围

仅统计满足以下条件的应收费用（CostItem）：

  • Cost.EntrustCd = CusContract.CusCd — 费用所属客户
  • CostItem.Ioid = 1
  • Amount > Ysje — 即存在未收金额
  • WhoName 不属于 Company 表中的公司名称
✅ 参与计算的是： 客户应收、尚未收齐、非公司主体 的费用。
NOT EXISTS ( SELECT 1 FROM Company WHERE Company.CusName = CostItem.WhoName )

2. 未收金额（Unpaid）

每笔费用：

未收金额 = Amount - Ysje

全部费用汇总：

Unpaid = Σ(Amount - Ysje) TotalAmount = Σ(Amount) TotalReceived = Σ(Ysje)

3. 可用授信额度

  • 正式授信： CreditAmount
  • 临时授信： TempCredit
    • 条件：TempType='C'，ApproveStatus='结束'，当前日期介于 EffDate ~ ExpDate
    • 若存在多条，TempCredit = ΣTempCredit
📌 可用授信 = CreditAmount + TempCredit

4. 统一授信规则（所有付款方式）

🔄 规则变更： 所有付款方式（包括现付 P、月结 M1/M2/M3/M4）不再区分，统一采用以下授信比较规则。

比较汇总未收金额 Unpaid 与可用授信 CreditAmount + TempCredit：

付款方式判定条件超额金额
所有方式
P M1 M2 M3 M4		 Unpaid > 可用授信 OverdraftAmount = Unpaid - 可用授信
若 ≤0 则无超额
✅ 现付（P）不再特殊处理，同样参与授信额度抵扣。仅当汇总未收金额超过可用授信总额时，才判定为超额。

二、超期（Overdue） 逐笔计算

超期针对 每一笔未收费用 分别计算。参与计算的数据同样要求 Amount > Ysje（存在未收金额）。

1. 获取账期类型

根据付款方式 PayWayCd 转换为月数：

付款方式代码月数
现付P0
月结M11
月结2M22
月结3M33
月结4M44
其它—0

记为 payMonths。

2. 账期天数 & 临时延期

  • 合同账期天数： AccDays（如 25 天）
  • 临时延期天数： 查询 CusCredit
    • 条件：TempType='A'，ApproveStatus='结束'，当前日期有效
    • 记为 TempCredit（此处表示延期天数，若不存在则为 0）
X = AccDays + TempCredit

3. 结算日（SettlementDate）计算

Step 1 — 计算 Y

if X > 30: Y = X + 30 else: Y = X + 30 × payMonths

将月结月份与账期天数统一换算为“总天数”。

Step 2 — 分解为月与日

if Y 能整除 30: settleMonths = Y/30 - 1 settleDays = 30 else: settleMonths = floor(Y/30) settleDays = Y % 30

Step 3 — 基准月份

以业务日期 BizDate 取该月第一天：

基准 = YYYY-MM-01

例如 BizDate = 2026-05-18 → 2026-05-01

Step 4 — 目标月份

目标 = 基准 + settleMonths 个月

例如 2026-05-01 + 2个月 = 2026-07-01

Step 5 — 结算日

  • 若 settleDays 超过目标月份的天数（如 31 日但二月只有 28 天），则取 月末最后一天。
  • 否则：
    结算日 = 目标月份第一天 + (settleDays - 1) 天

例如：7月，settleDays=20 → 7月20日

4. 超期判定

超期天数 = 今天 - 结算日

SQL 实现：

DATEDIFF(DAY, SettlementDate, GETDATE())
⚠️ 若 超期天数 > 0，则 Overdue = 1（超期）；否则 Overdue = 0。
📌 最终 SQL 只返回 OverdueDays > 0 的数据，即只有已经超过结算日期的未收费用才会进入超期结果。

三、整体流程 概览

费用（CostItem） │ ▼ 过滤： Ioid=1 未收金额 > 0 非 Company 主体 │ ▼ 统计全部未收金额 (Unpaid) │ ▼ Unpaid > 正式授信 + 临时授信 ? │ ├─ 是 → 超额 (Overdraft=1) 金额 = Unpaid - 授信 └─ 否 → 正常 (Overdraft=0) │ ▼ 超期计算（费用明细逐笔） │ BizDate │ 付款方式（月数: P=0, M1=1, M2=2, M3=3, M4=4） │ AccDays │ TempType = A 延期天数 │ ▼ 计算 SettlementDate │ ▼ 今天 > SettlementDate ? │ ├─ 是 → Overdue = 1 └─ 否 → Overdue = 0 ⚡ 变更点：现付（P）不再单独分支，所有付款方式统一参与授信比较；代码规范为 M1/M2/M3/M4。
🧩 关键差异 · 超额采用 客户维度汇总 计算（所有未收金额汇总后与授信比较）；超期采用 费用明细维度逐笔 计算（每笔费用独立计算结算日）。两部分的计算粒度不同，是当前实现的重要特点。
