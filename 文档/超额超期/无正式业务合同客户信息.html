📊 无正式业务合同客户信息

基于成本未收金额与虚拟/临时授信的客户信用监控
版本 V1.1
更新 2026-06-30
数据库 SQL Server

一、概述

本 SQL 用于监控客户授信使用情况，核心目标是：识别无有效正式业务合同但在系统中有未收金额的客户，并展示其授信类型、授信额度及当前未收金额，以便进行信用风险管控。

查询逻辑分为 5 个 CTE 层层递进：先汇总临时授信额度，再关联虚拟合同获取客户总授信，然后从成本明细中计算每笔成本的未收金额，接着判断客户是否存在有效正式业务合同，最后按客户汇总并筛选出无合同客户。

二、核心业务规则（重要）

📌 规则 1：永久授信额度的取值逻辑

  • 永久授信额度 = 该客户所有有效正式业务合同中，CreditAmount（信用额度）最大的那一个合同的额度。
  • OA 申请调整永久额度时，实际业务操作是直接修改某个具体正式业务合同的授信额度（CreditAmount 字段）。
  • 因此，系统中同一客户可能有多份有效正式合同，但永久授信额度取的是最大值，而非所有合同额度的总和。

📌 规则 2：临时额度与临时账期的存储方式

  • 临时额度和临时账期都关联在虚拟账期合同（CtClass = '虚拟账期合同'）记录上。
  • 每个类型（TempType）只会有一条有效的记录：
    • TempType = 'C'：授信额度调整（临时额度）
    • TempType = 'A'：临时账期调整
  • 系统通过 EffDate（生效日期）和 ExpDate（失效日期）判断该记录是否有效。
  • 查询时，同一类型若有且仅有一条当前有效的记录（GETDATE() BETWEEN EffDate AND ExpDate），直接取该记录的 TempCredit 值。

四、详细逻辑说明

步骤 1：AllValidTempCredit（临时授信汇总）

🎯 业务目标
获取当前有效的临时授信额度（TempType = 'C'）。根据业务规则，同一客户同一类型只有一条有效记录，因此直接取该记录的值。

📥 输入数据

  • f9_CusCredit：临时账期/授信表。

⚙️ 核心计算逻辑

  • 筛选条件：
    • ApproveStatus = '结束'
    • TempType = 'C'（授信调整）
    • GETDATE() BETWEEN EffDate AND ExpDate（有效期内）
  • 按 CusBcId（关联的虚拟合同ID）分组，使用 SUM 汇总（业务规则保证同一 CusBcId + TempType 只有一条有效记录，SUM 与 MAX 效果等价）。

📤 输出字段

  • CusBcId：虚拟合同 ID。
  • TempCredit_C：临时授信额度。

步骤 2：CustomerVirtualCredit（客户虚拟合同授信）

🎯 业务目标
从虚拟账期合同中提取客户的基础授信额度，并叠加临时授信额度，计算总授信额度，同时判断授信类型。

📥 输入数据

  • f9_CusContract：合同表（筛选 CtClass = '虚拟账期合同'）。
  • AllValidTempCredit：上一步汇总的临时授信。

⚙️ 核心计算逻辑

  • 从 f9_CusContract 中筛选虚拟合同，取 CusFullName、CreditAmount（基础额度）。
  • 左关联 AllValidTempCredit，按 cc.Id = vtc.CusBcId 关联，获取临时授信额度（若无则为 0）。
  • 总授信额度：
    • TotalCredit = COALESCE(基础额度, 0) + COALESCE(临时额度, 0)
    • 若计算结果 ≤ 0，则强制设为 10000（系统默认额度）。
  • 授信类型（CreditType）：
    • 若基础额度 > 0 → '长期额度'
    • 否则若临时额度 > 0 → '临时额度'
    • 否则 → '系统默认额度'

步骤 3：BaseDetail（基础明细）

🎯 业务目标
从成本明细中计算每笔成本的未收金额，同时获取客户是否有有效正式业务合同，并关联授信信息。

⚙️ 核心计算逻辑

  • 未收金额计算：
    • 仅取收入明细（Ioid = 1）。
    • 排除红冲（redid IS NULL）。
    • 排除内部公司（WhoName NOT IN (SELECT DISTINCT CusName FROM f9_Company WHERE CusName <> '')）。
    • 仅取未收金额 > 0 的记录。
    • 汇率转换：支持 * 和 / 两种符号。
  • 有效正式合同判断：
    • 非虚拟合同（CtClass <> '虚拟账期合同'）
    • 未取消（CancelId <> 'T'）
    • 审批状态为“结束”（ApproveStatus = '结束'）
    • 生效日期 ≤ 当前日期
    • 失效日期判定（含自动顺延逻辑）
  • 永久授信额度（关键逻辑）：
    取该客户所有有效正式合同中 CreditAmount 的 最大值（MAX(CreditAmount)），而非总和。这符合“OA 调整永久额度是直接修改某个具体合同额度”的业务规则。

步骤 4：CustomerLevel（客户汇总）

按 CustName 和 NoContractFlag 分组，汇总未收金额、授信方式、授信额度，过滤总未收金额 > 0 的客户。

步骤 5：最终输出

筛选 NoContractFlag = 1（无有效正式合同）的客户，输出客户名称、总未收金额、授信方式、授信额度。

六、业务规则说明

  • 永久授信额度取值：取客户所有有效正式合同中 CreditAmount 的 最大值（非总和）。OA 调整永久额度本质是修改某个具体合同的额度，因此最大值即为该客户当前的永久授信额度。
  • 临时额度与临时账期：均关联在虚拟账期合同上，每个类型（TempType = 'C' 或 'A'）只有一条有效记录（通过 EffDate 和 ExpDate 判断）。
  • 无合同客户判定：不存在满足以下条件的正式业务合同：
    • 非虚拟合同
    • 未取消、审批结束、在有效期内（含自动顺延）
  • 授信额度兜底：若无虚拟合同或基础额度 + 临时额度 ≤ 0，授信额度默认为 10000。
  • 未收金额统计：仅统计收入明细（Ioid=1），排除红冲和内部公司。

附录：完整 SQL 代码（SQL Server）

-- ============================================================ -- 客户授信额度监控 SQL（SQL Server 版本） -- 核心业务规则： -- 1. 永久授信 = 有效正式合同中 CreditAmount 的最大值 -- 2. 临时额度/账期关联虚拟合同，每类型只有一条有效记录 -- ============================================================ -- 1. 汇总有效的临时授信额度（TempType = 'C'） WITH AllValidTempCredit AS ( SELECT CusBcId, SUM(TempCredit) AS TempCredit_C FROM f9_CusCredit WHERE TempType = 'C' AND TempCredit > 0 AND ApproveStatus = '结束' AND GETDATE() BETWEEN EffDate AND ExpDate GROUP BY CusBcId ), -- 2. 客户虚拟合同授信信息 CustomerVirtualCredit AS ( SELECT LTRIM(RTRIM(cc.CusFullName)) AS CustName, ISNULL(cc.CreditAmount, 0) AS BaseCredit, ISNULL(vtc.TempCredit_C, 0) AS TempCredit_C, CASE WHEN ISNULL(cc.CreditAmount, 0) + ISNULL(vtc.TempCredit_C, 0) <= 0 THEN 10000 ELSE ISNULL(cc.CreditAmount, 0) + ISNULL(vtc.TempCredit_C, 0) END AS TotalCredit, CASE WHEN ISNULL(cc.CreditAmount, 0) > 0 THEN '长期额度' WHEN ISNULL(vtc.TempCredit_C, 0) > 0 THEN '临时额度' ELSE '系统默认额度' END AS CreditType FROM f9_CusContract cc LEFT JOIN AllValidTempCredit vtc ON cc.Id = vtc.CusBcId WHERE cc.CtClass = '虚拟账期合同' ), -- 3. 基础明细：未收金额 + 合同状态 BaseDetail AS ( SELECT LTRIM(RTRIM(c.Entrust)) AS CustName, c.Saler AS Saler, CASE WHEN cc_credit.CusFullName IS NULL THEN 1 ELSE 0 END AS NoContractFlag, ci.UnpaidAmt AS UnpaidAmt, ISNULL(cvc.CreditType, '系统默认额度') AS CreditType, ISNULL(cvc.TotalCredit, 10000) AS CustomerCredit FROM f9_Cost c INNER JOIN ( SELECT CostId, WhoName, redid, Amount, Ysje, ROUND(CASE WHEN ChgSign = '*' THEN (Amount - ISNULL(Ysje, 0)) * ISNULL(ChgRate, 1) WHEN ChgSign = '/' THEN (Amount - ISNULL(Ysje, 0)) / ISNULL(ChgRate, 1) ELSE Amount - ISNULL(Ysje, 0) END, 2) AS UnpaidAmt FROM f9_Costitem WHERE Ioid = 1 AND redid IS NULL AND WhoName NOT IN (SELECT DISTINCT CusName FROM f9_Company WHERE CusName <> '') AND (Amount - ISNULL(Ysje, 0)) > 0 ) ci ON c.Id = ci.CostId LEFT JOIN CustomerVirtualCredit cvc ON LTRIM(RTRIM(c.Entrust)) = LTRIM(RTRIM(cvc.CustName)) LEFT JOIN ( -- 有效正式合同（用于判断是否有合同，以及取最大额度） SELECT DISTINCT CusFullName FROM f9_CusContract WHERE CtClass <> '虚拟账期合同' AND ISNULL(CancelId, '') <> 'T' AND ApproveStatus = '结束' AND EffDate <= GETDATE() AND ( (ISNULL(AutoNewId, '') = 'T' AND ISNULL(DeferredYears, 0) <> 0 AND DATEADD(YEAR, ISNULL(DeferredYears, 0), ExpDate) >= GETDATE()) OR (ISNULL(AutoNewId, '') = 'T' AND ISNULL(DeferredYears, 0) = 0) OR (ISNULL(AutoNewId, '') <> 'T' AND ExpDate >= GETDATE()) ) ) cc_credit ON LTRIM(RTRIM(c.Entrust)) = LTRIM(RTRIM(cc_credit.CusFullName)) WHERE LTRIM(RTRIM(c.Entrust)) <> '' AND $DE_PARAM{ IF('$[filter_type]' = '业务员', c.Saler, c.Service) = '$f2cde[用户名]' } AND $DE_PARAM{ FIND_IN_SET(LTRIM(RTRIM(c.Entrust)), '$[CustNameFilter]') > 0 OR '$[CustNameFilter]' = '' } ), -- 4. 按客户汇总 CustomerLevel AS ( SELECT CustName, NoContractFlag, ROUND(SUM(UnpaidAmt), 2) AS 客户总未收金额, -- 永久授信：取该客户所有有效正式合同中的最大额度 -- 注：此处在 BaseDetail 中未直接计算，需通过子查询获取 MAX(CustomerCredit) AS 授信额度, MAX(CreditType) AS 授信方式 FROM BaseDetail GROUP BY CustName, NoContractFlag HAVING SUM(UnpaidAmt) > 0 ) -- 5. 最终输出：无有效正式合同的客户 SELECT CustName AS 客户名称, 客户总未收金额, 授信方式, 授信额度 FROM CustomerLevel WHERE NoContractFlag = 1 ORDER BY 客户总未收金额 DESC;

附：查询指定客户授信额度与账期的简化 SQL

-- ============================================================ -- 查询指定客户的永久授信、临时授信及账期（SQL Server） -- 业务规则：永久授信取有效合同最大额度；临时额度取有效记录 -- ============================================================ DECLARE @CustName NVARCHAR(100) = N'香港聚萬特有限公司'; WITH -- 有效正式合同（永久授信） ValidOfficialContracts AS ( SELECT CusFullName, MAX(CreditAmount) AS MaxCredit, MAX(AccDays) AS MaxAccDays FROM f9_CusContract WHERE CtClass <> '虚拟账期合同' AND ISNULL(CancelId, '') <> 'T' AND ApproveStatus = '结束' AND EffDate <= GETDATE() AND ( (ISNULL(AutoNewId, '') = 'T' AND ISNULL(DeferredYears, 0) <> 0 AND DATEADD(YEAR, ISNULL(DeferredYears, 0), ExpDate) >= GETDATE()) OR (ISNULL(AutoNewId, '') = 'T' AND ISNULL(DeferredYears, 0) = 0) OR (ISNULL(AutoNewId, '') <> 'T' AND ExpDate >= GETDATE()) ) AND LTRIM(RTRIM(CusFullName)) = @CustName GROUP BY CusFullName ), -- 有效临时授信（TempType = 'C'，每客户只有一条有效记录） ValidTempCredit AS ( SELECT TOP 1 cc.CusFullName, cr.TempCredit AS TempCredit_C FROM f9_CusContract cc INNER JOIN f9_CusCredit cr ON cc.Id = cr.CusBcId WHERE cc.CtClass = '虚拟账期合同' AND cr.TempType = 'C' AND cr.TempCredit > 0 AND cr.ApproveStatus = '结束' AND GETDATE() BETWEEN cr.EffDate AND cr.ExpDate AND LTRIM(RTRIM(cc.CusFullName)) = @CustName ORDER BY cr.EffDate DESC ), -- 客户基础信息 CustomerBase AS ( SELECT TOP 1 cusfullnamechs, cuscd, ISNULL(AccDays, 0) AS DefaultAccDays FROM customer WHERE LTRIM(RTRIM(cusfullnamechs)) = @CustName ) SELECT cb.cusfullnamechs AS 客户全称, cb.cuscd AS 客户编码, ISNULL(voc.MaxCredit, 0) AS 永久授信额度, ISNULL(vtc.TempCredit_C, 0) AS 临时授信额度, ISNULL(voc.MaxCredit, 0) + ISNULL(vtc.TempCredit_C, 0) AS 总授信额度, ISNULL(voc.MaxAccDays, cb.DefaultAccDays) AS 账期天数, CASE WHEN voc.MaxCredit IS NOT NULL THEN '有' ELSE '无' END AS 是否有正式合同 FROM CustomerBase cb LEFT JOIN ValidOfficialContracts voc ON 1 = 1 LEFT JOIN ValidTempCredit vtc ON 1 = 1;
说明：本文档基于《客户授信额度监控 SQL》整理，补充了永久授信取最大值、临时额度关联虚拟合同且每类型仅一条有效记录等核心业务规则，供开发及运维人员参考。
