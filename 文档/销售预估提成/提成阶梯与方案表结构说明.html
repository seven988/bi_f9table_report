Zj_CommissionScheme & Zj_CommissionLadder 字段、关联及业务场景
版本 V1.1
更新 2026-06-29
数据库 SQL Server

一、概述

在提成计算系统中，提成方案表（Zj_CommissionScheme）和提成阶梯表（Zj_CommissionLadder）是核心的基础配置表。方案表定义了“谁”适用“什么规则”（如业务员、客服、公司/个人业务），阶梯表则定义了“具体金额区间”对应“多少提成比例”。两者通过 Ladder_Id 字段关联，共同构成完整的提成规则体系。

本说明文档详细列出了两张表的所有字段、数据类型、业务含义及示例值，并说明了它们在 销售提成（单笔匹配） 和 客服提成（累计分段） 两种场景下的不同用法。

二、提成方案表（Zj_CommissionScheme）

该表存储提成方案的头部信息，包括方案名称、适用人员类型（业务员/客服）、计算方式（单笔/累计）、适用业务大类、适用接单公司列表、有效期等。

字段名 数据类型 含义 示例值 备注
Ladder_Id INT 阶梯方案ID 1001 外键，关联 Zj_CommissionLadder
SchemeName NVARCHAR(100) 方案名称 华东区销售提成方案 便于识别
SchemeType VARCHAR(20) 方案类型 SALER / SERVICE SALER 业务员 SERVICE 客服
CalcType VARCHAR(20) 计算方式 SINGLE / CUMULATIVE SINGLE 单笔匹配 CUMULATIVE 累计分段
BusinessType NVARCHAR(50) 业务大类 跨境电商 匹配 Cost.BusWay 或转换后的业务大类
IsCompanyBusiness BIT 公司/个人业务标识 1（公司业务） 客服提成专用，1=公司业务，0=个人业务
CtrlOffice NVARCHAR(500) 适用接单公司列表 上海分公司,宁波分公司 逗号分隔，支持多个公司
Status BIT 启用状态 1 1=启用，0=停用
EffectiveDate DATETIME 生效日期 2026-01-01 NULL 表示无限制
ExpirationDate DATETIME 失效日期 2026-12-31 NULL 表示无限制

三、提成阶梯表（Zj_CommissionLadder）

该表存储具体的阶梯档位，每个阶梯属于某个方案（Ladder_Id），包含金额区间（MinAmount ~ MaxAmount）和对应的提成比例（CommissionRate），以及排序序号。

字段名 数据类型 含义 示例值 备注
Ladder_Id INT 阶梯方案ID 1001 外键，关联方案表
LadderSeq INT 阶梯序号 1, 2, 3 用于排序，决定匹配优先级
LadderName NVARCHAR(50) 阶梯名称 高档 便于识别，如“低档”、“中档”、“高档”
MinAmount DECIMAL(18,2) 区间最小金额 0.00 含（左闭），NULL 视为 0
MaxAmount DECIMAL(18,2) 区间最大金额 10000.00 不含（右开），NULL 视为无穷大
CommissionRate DECIMAL(10,4) 提成比例 0.0500 小数形式，0.05 表示 5%

四、表关联关系

关联字段：Zj_CommissionScheme.Ladder_Id → Zj_CommissionLadder.Ladder_Id（一对多关系，一个方案可包含多个阶梯）
graph LR A[Zj_CommissionScheme
方案表] -->|Ladder_Id| B[Zj_CommissionLadder
阶梯表] A -->|SchemeType| C(区分: SALER / SERVICE) A -->|CalcType| D(区分: SINGLE / CUMULATIVE) B -->|MinAmount / MaxAmount| E(匹配毛利区间) E --> F(计算提成金额)

通过 Ladder_Id 关联方案表与阶梯表，SchemeType 和 CalcType 决定了提成的业务类型与计算方式。

五、业务场景使用说明

5.1 销售提成（业务员）

  • 方案筛选：SchemeType = 'SALER' 且 CalcType = 'SINGLE'。
  • 匹配方式：对每一笔毛利，使用 ABS(毛利金额) 匹配阶梯表中 MinAmount ≤ 金额 ＜ MaxAmount 的唯一阶梯，取出 CommissionRate 作为提成比例。
  • 负毛利处理：若毛利为负，直接使用该方案下阶梯表中的最大比例（MAX(CommissionRate)）作为比例。
  • 额外标识：方案表中 IsCompanyBusiness 字段不用于销售提成（该字段仅客服提成使用）。

5.2 客服提成

  • 方案筛选：SchemeType = 'SERVICE' 且 CalcType = 'CUMULATIVE'。
  • 匹配方式：先根据业务员姓名判断“公司业务”或“个人业务”，匹配方案中的 IsCompanyBusiness 字段；然后对月度汇总后的毛利金额进行累计分段计算：将毛利按绝对值逐段落在每个阶梯区间内，每段乘以对应的 CommissionRate，最后累加得到提成金额（正毛利为正，负毛利为负）。
  • 关键字段：方案表中的 IsCompanyBusiness 用于区分公司/个人业务，分别匹配不同的提成方案。

5.3 海外仓提成

  • 海外仓业务本质上与销售提成相同，使用 SchemeType = 'SALER' 和 CalcType = 'SINGLE'，只是毛利按季度汇总，且业务大类固定为“海外仓”。
  • 方案表需配置 BusinessType = '海外仓' 的方案，匹配规则与销售提成一致。

六、注意事项

  • 阶梯区间边界：MinAmount 为包含（≥），MaxAmount 为不包含（＜），请确保各阶梯区间连续且不重叠，避免数据匹配歧义。
  • NULL 处理：MinAmount 为 NULL 时视为 0；MaxAmount 为 NULL 时视为无穷大，通常用于最高档阶梯。
  • 方案有效期：EffectiveDate 和 ExpirationDate 支持 NULL，表示无限制。在 SQL 中判断当前日期是否在有效期内需考虑 NULL 情况。
  • 公司列表拆分：CtrlOffice 字段用逗号分隔，在 SQL 中通过 XML 解析拆分成多行，每个公司独立匹配。
  • 客服提成特殊字段：IsCompanyBusiness 仅在客服提成方案中有意义，销售提成方案中该字段可为 NULL 或默认值，不影响匹配。
说明：本文档仅涵盖与提成计算直接相关的核心字段，实际表中可能包含其他辅助字段（如创建时间、修改人、备注等），具体请参考数据库设计文档。
