按季度汇总海外仓毛利 & 阶梯方案业务员提成核算

数据库 SQL Server

一、概述

本 SQL 专门用于计算 海外仓业务 的销售提成。与普通销售提成不同，海外仓业务采用 季度维度 进行毛利汇总，业务大类固定为「海外仓」。核心逻辑为：从成本明细中按季度聚合毛利，按接单公司和业务大类匹配提成方案，再根据毛利金额所在的阶梯区间计算提成比例，最终得出提成金额。整个计算包含 5 个逻辑层（CTE）。

三、详细逻辑说明

步骤 1：海外仓毛利数据（CTE: 海外仓毛利数据）

🎯 业务目标
筛选业务大类为「海外仓」的成本单，按 季度 汇总毛利，并生成季度显示字符串和航期月（当季最后一个月），为后续按季度提成做准备。

📥 输入数据

  • Cost（成本单头）：提供接单公司、业务员、业务日期、业务大类。
  • CostItem（成本明细）：提供收支标识（Ioid）、金额、汇率及运算符号。

⚙️ 核心计算逻辑

  • 毛利 = 收入合计 - 支出合计，每行明细按 ChgSign（'*' 或 '/'）与 ChgRate 进行汇率折算，得到本币金额。
  • 固定业务大类：直接赋值为 '海外仓'（c.BusWay = '海外仓' 过滤）。
  • 季度字符串：例如“2026年1-3月”，由年份 + 季度起始月 + 结束月拼接。
  • 航期月：当季最后一个月，格式为 YYYY-MM，例如“2026-03”，用于后续可能的月度核算。

📤 输出字段
接单公司, 实际业务大类（固定'海外仓'）, 业务员, 季度, 航期月, 毛利金额。

⚠️ 过滤条件

  • 日期范围（$DE_PARAM）
  • 未取消（Cancelid <> 'T'）
  • 业务员非空
  • 非红冲明细（redid IS NULL）
  • 业务大类 = '海外仓'

💡 注意：分组维度为 ctrloffice, Saler, YEAR(BizDate), DATEPART(QUARTER, BizDate)，即每个公司+业务员+季度只保留一条汇总记录。

步骤 2：方案拆分（CTE: 方案拆分）

🎯 业务目标
与普通销售提成相同，从提成方案表中将“接单公司”字段（逗号分隔）拆分成多行，筛选出针对业务员（SALER）、单笔计算（SINGLE）、启用的方案。

📥 输入数据
Zj_CommissionScheme（提成方案表）。

⚙️ 核心逻辑

  • 使用 CROSS APPLY + XML 解析拆分 CtrlOffice。
  • 过滤条件：SchemeType = 'SALER', CalcType = 'SINGLE', Status = 1, 当前日期在有效期内。

📤 输出字段
Ladder_Id, SchemeName, BusinessType, 接单公司（拆分行）。

步骤 3：方案匹配（CTE: 方案匹配）

🎯 业务目标
将海外仓毛利数据与拆分后的方案进行关联，匹配条件为 接单公司 + 实际业务大类（即'海外仓'），去重后获得阶梯 ID。

📥 输入数据
来自 海外仓毛利数据 和 方案拆分。

⚙️ 核心逻辑

  • 关联条件：m.接单公司 = sm.接单公司 AND m.实际业务大类 = sm.BusinessType。
  • 使用 DISTINCT 去重。

📤 输出字段
接单公司, BusinessType, Ladder_Id, SchemeName。

步骤 4：提成计算（CTE: 提成计算）

🎯 业务目标
根据匹配到的阶梯方案，计算每个季度毛利记录对应的提成比例（正毛利匹配区间，负毛利使用最高比例）。

📥 输入数据
来自 海外仓毛利数据 和 方案匹配，以及 Zj_CommissionLadder（阶梯表）。

⚙️ 核心计算逻辑

  • 最高提成比例：该 Ladder_Id 下最大 CommissionRate。
  • 匹配提成比例：使用 ABS(毛利金额) 匹配满足 MinAmount ≤ 毛利 ＜ MaxAmount 的阶梯，按 LadderSeq 取第一条。
  • 若未找到匹配阶梯，则匹配提成比例为 NULL，后续使用最高比例作为默认。

📤 输出字段
所有毛利维度字段 + Ladder_Id, SchemeName, 最高提成比例, 匹配提成比例。

💡 注意：负毛利直接使用最高比例，与普通销售提成规则一致。

步骤 5：最终输出（SELECT）

🎯 业务目标
生成最终提成报表，包含季度、航期月、提成比例（带状态标注）和提成金额，并增加固定字段（费用类型、业务类型）和预估年月。

📥 输入数据
来自 提成计算 CTE。

⚙️ 输出字段计算

  • 提成比例：逻辑同销售提成，未匹配、负毛利、正毛利分别显示不同文本。
  • 提成金额：未匹配方案为 0；负毛利用最高比例；正毛利用匹配比例或最高比例兜底。
  • 预估年月：当前系统日期（GETDATE()）的 yyyy-MM。
  • 费用类型：固定为“销售费用”。
  • 业务类型：空字符串（保留字段）。

📤 最终输出
一张按季度展示的海外仓提成明细表，便于季度考核与核对。

五、业务规则说明

  • 海外仓业务固定大类：所有记录的业务大类硬编码为「海外仓」，方案匹配时需保证方案中 BusinessType 也为「海外仓」。
  • 季度汇总维度：按 接单公司 + 业务员 + 季度 聚合，同一季度内所有成本单的毛利加总。
  • 提成比例匹配规则：正毛利按 ABS(毛利金额) 匹配阶梯区间；负毛利直接使用该阶梯的 最高提成比例。
  • 未匹配方案处理：若未匹配到任何提成方案，则提成金额为 0，提成比例显示「未设置提成规则」。
  • 数据过滤：过滤已取消、红冲、业务员为空、业务大类非「海外仓」的记录。
  • 日期参数：使用 $DE_PARAM{...} 动态传入 BEGINTIME 和 ENDTIME。

-- ============================================================ -- 海外仓提成计算 SQL（完整版） -- 数据库：SQL Server -- ============================================================ -- 1. 海外仓毛利数据（按季度汇总） WITH 海外仓毛利数据 AS ( SELECT RTRIM(LTRIM(c.ctrloffice)) AS 接单公司, '海外仓' AS 实际业务大类, c.Saler AS 业务员, CAST(YEAR(c.BizDate) AS VARCHAR(4)) + '年' + CAST(DATEPART(QUARTER, c.BizDate) * 3 - 2 AS VARCHAR(2)) + '-' + CAST(DATEPART(QUARTER, c.BizDate) * 3 AS VARCHAR(2)) + '月' AS 季度, CAST(YEAR(c.BizDate) AS VARCHAR(4)) + '-' + RIGHT('0' + CAST(DATEPART(QUARTER, c.BizDate) * 3 AS VARCHAR(2)), 2) AS 航期月, ROUND( SUM(CASE WHEN ci.Ioid = 1 THEN CASE WHEN ci.ChgSign = '*' THEN ci.Amount * ISNULL(ci.ChgRate, 1) WHEN ci.ChgSign = '/' THEN ci.Amount / ISNULL(ci.ChgRate, 1) ELSE ci.Amount END ELSE 0 END) - SUM(CASE WHEN ci.Ioid = -1 THEN CASE WHEN ci.ChgSign = '*' THEN ci.Amount * ISNULL(ci.ChgRate, 1) WHEN ci.ChgSign = '/' THEN ci.Amount / ISNULL(ci.ChgRate, 1) ELSE ci.Amount END ELSE 0 END), 2) AS 毛利金额 FROM Cost c INNER JOIN CostItem ci ON c.Id = ci.CostId WHERE 1=1 AND $DE_PARAM{ c.BizDate >= '$[BEGINTIME]' AND c.BizDate < DATEADD(DAY, 1, '$[ENDTIME]') } AND c.Cancelid <> 'T' AND c.Saler IS NOT NULL AND c.Saler <> '' AND ci.redid IS NULL AND c.BusWay = '海外仓' GROUP BY c.ctrloffice, c.Saler, YEAR(c.BizDate), DATEPART(QUARTER, c.BizDate) ), -- 2. 拆分提成方案（只取业务提成） 方案拆分 AS ( SELECT s.Ladder_Id, s.SchemeName, s.BusinessType, LTRIM(RTRIM(Split.a.value('.', 'NVARCHAR(MAX)'))) AS 接单公司 FROM Zj_CommissionScheme s CROSS APPLY (SELECT CAST('<M>' + REPLACE(s.CtrlOffice, ',', '</M><M>') + '</M>' AS XML) AS x) t CROSS APPLY t.x.nodes('/M') Split(a) WHERE s.SchemeType = 'SALER' AND s.CalcType = 'SINGLE' AND s.CtrlOffice IS NOT NULL AND s.CtrlOffice != '' AND s.Status = 1 AND (s.EffectiveDate <= GETDATE() OR s.EffectiveDate IS NULL) AND (s.ExpirationDate >= GETDATE() OR s.ExpirationDate IS NULL) ), -- 3. 方案匹配（去重） 方案匹配 AS ( SELECT DISTINCT s.接单公司, s.BusinessType, s.Ladder_Id, s.SchemeName FROM 方案拆分 s ), -- 4. 提成计算 提成计算 AS ( SELECT m.接单公司, m.实际业务大类, m.业务员, m.季度, m.航期月, m.毛利金额, sm.Ladder_Id, sm.SchemeName, (SELECT MAX(CommissionRate) FROM Zj_CommissionLadder l WHERE l.Ladder_Id = sm.Ladder_Id) AS 最高提成比例, (SELECT TOP 1 l.CommissionRate FROM Zj_CommissionLadder l WHERE l.Ladder_Id = sm.Ladder_Id AND (l.MinAmount IS NULL OR ABS(m.毛利金额) >= l.MinAmount) AND (l.MaxAmount IS NULL OR ABS(m.毛利金额) < l.MaxAmount) ORDER BY l.LadderSeq) AS 匹配提成比例 FROM 海外仓毛利数据 m LEFT JOIN 方案匹配 sm ON m.接单公司 = sm.接单公司 AND m.实际业务大类 = sm.BusinessType ) -- 5. 最终输出 SELECT t.接单公司, t.实际业务大类, t.业务员, t.季度, t.航期月, '' AS 业务类型, FORMAT(GETDATE(), 'yyyy-MM') AS 预估年月, '销售费用' AS 费用类型, t.毛利金额, CASE WHEN t.Ladder_Id IS NULL THEN '未设置提成规则' WHEN t.毛利金额 < 0 THEN CONVERT(VARCHAR(50), ROUND(ISNULL(t.最高提成比例, 0) * 100, 2)) + '% (负数毛利)' ELSE ISNULL( CONVERT(VARCHAR(50), ROUND(t.匹配提成比例 * 100, 2)) + '%', CONVERT(VARCHAR(50), ROUND(ISNULL(t.最高提成比例, 0) * 100, 2)) + '% (默认)' ) END AS 提成比例, CASE WHEN t.Ladder_Id IS NULL THEN 0 WHEN t.毛利金额 < 0 THEN ROUND(t.毛利金额 * ISNULL(t.最高提成比例, 0), 2) ELSE ROUND(t.毛利金额 * ISNULL(t.匹配提成比例, ISNULL(t.最高提成比例, 0)), 2) END AS 提成金额 FROM 提成计算 t ORDER BY t.接单公司, t.实际业务大类, t.业务员, t.季度;
