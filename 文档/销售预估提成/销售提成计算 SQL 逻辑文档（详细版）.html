基于成本毛利 & 阶梯方案的业务员提成核算
版本 V2.1
更新 2026-06-29
数据库 SQL Server

一、概述

本 SQL 用于计算业务员在指定时间段内的 销售提成。 核心逻辑是：从成本明细中汇总毛利，按业务大类匹配提成方案，再根据毛利金额所在的阶梯区间计算提成比例，最终得出提成金额。 整个计算过程包含 6 个逻辑层（CTE），层层递进。

三、详细逻辑说明

步骤 1：成本毛利（CTE: 成本毛利）

🎯 业务目标
在 CostId 粒度上，汇总每一张成本单的净毛利（收入 - 支出），并同时获取该成本单对应的分单状态、主单业务大类等属性，为后续业务大类转换做准备。

📥 输入数据

  • Cost（成本单头）：提供接单公司、业务员、业务日期、原始业务大类。
  • CostItem（成本明细）：提供收支标识（Ioid）、金额、汇率及运算符号。
  • Export（出口单）：通过 LEFT JOIN 关联，获取当前单是否分单（SplitMblId）以及主单的业务大类（e_main.BusWay）。

⚙️ 核心计算逻辑

  • 毛利 = 收入合计 - 支出合计，其中每一行明细均按 ChgSign（'*' 或 '/'）与 ChgRate 进行汇率折算，得到本币金额。
  • 分单识别：若 e_self.SplitMblId IS NOT NULL 则标记为分单（1），否则非分单（0）。
  • 主单业务大类：通过关联 e_main.SplitId = 'P' 取出主单的 BusWay，供后续代拉代报转换使用。

📤 输出字段
CostId, 接单公司, 业务员, 月度, 原始业务大类, 主单业务大类, 是否分单, 毛利金额。

⚠️ 重要过滤条件

  • 仅统计指定日期区间内的业务（$DE_PARAM）。
  • 排除已取消（Cancelid = 'T'）、红冲明细（redid IS NOT NULL）、业务员为空、业务大类为空或为“海外仓”的记录。

💡 注意：该 CTE 以 CostId 为最小分组单位，确保每一张单独立计算毛利，不会因后续聚合而混淆。

步骤 2：毛利数据（CTE: 毛利数据）

🎯 业务目标
对上一步得到的“原始业务大类”进行业务转换，生成“实际业务大类”，并将同一成本单下可能存在的多行（理论上前一步已按 CostId 聚合，但此处仍做二次聚合以保证一致性）按维度再次汇总。

📥 输入数据
来自 成本毛利 CTE 的输出。

⚙️ 核心转换规则

  • 代拉代报转换：若原始业务大类为“代拉代报”：
    • 如果该单是分单（是否分单 = 1）且主单业务大类不为空，则实际业务大类 = 主单业务大类。
    • 否则保持“代拉代报”不变。
  • 跨境电商合并：将“跨境电商头程”、“跨境电商尾程”、“跨境电商全段”统一归并为“跨境电商”。
  • 其他大类：保持原值。

📤 输出字段
接单公司, 实际业务大类, 业务员, 月度, 毛利金额（按上述维度求和）。

💡 注意：GROUP BY 中重复了完整的 CASE 表达式，这是 SQL Server 的要求（不能直接引用别名），确保分组正确。

步骤 3：方案拆分（CTE: 方案拆分）

🎯 业务目标
从提成方案表中，将“接单公司”字段（以逗号分隔的多公司）拆分成多行，同时过滤出符合当前时间范围的、启用的、针对业务员（SALER）且为单笔计算（SINGLE）的方案。

📥 输入数据
Zj_CommissionScheme（提成方案表）。

⚙️ 核心逻辑

  • 使用 CROSS APPLY + XML 解析，将 CtrlOffice 按逗号拆分为多行。
  • 过滤条件：
    • SchemeType = 'SALER'（业务员提成）
    • CalcType = 'SINGLE'（单笔计算，非累计）
    • Status = 1（启用）
    • 当前日期在 EffectiveDate 和 ExpirationDate 范围内（NULL 表示无限制）。

📤 输出字段
Ladder_Id, SchemeName, BusinessType, 接单公司（拆分行）。

💡 注意：同一方案若包含多个公司，拆分后每个公司一行，后续匹配时每个公司独立匹配。

步骤 4：方案匹配（CTE: 方案匹配）

🎯 业务目标
将毛利数据与拆分后的方案进行关联，为每一组（接单公司 + 实际业务大类）找到对应的提成方案（阶梯 ID 和名称），并去重，避免重复匹配。

📥 输入数据
来自 毛利数据 和 方案拆分。

⚙️ 核心逻辑

  • 关联条件：毛利数据.接单公司 = 方案拆分.接单公司 且 毛利数据.实际业务大类 = 方案拆分.BusinessType。
  • 使用 DISTINCT 去除重复（因为一个方案可能有多行，但关联后只取一次）。

📤 输出字段
接单公司, BusinessType, Ladder_Id, SchemeName。

💡 注意：若某组（公司+大类）未匹配到任何方案，则后续提成金额为 0。

步骤 5：提成计算（CTE: 提成计算）

🎯 业务目标
根据匹配到的阶梯方案，计算每个毛利记录对应的提成比例（正毛利匹配区间，负毛利使用最高比例），并保留最高比例作为兜底。

📥 输入数据
来自 毛利数据 和 方案匹配，以及 Zj_CommissionLadder（阶梯表）。

⚙️ 核心计算逻辑

  • 最高提成比例：取该 Ladder_Id 下所有阶梯中最大的 CommissionRate（用于负数毛利和未匹配到具体阶梯时）。
  • 匹配提成比例：使用 ABS(毛利金额) 匹配满足 MinAmount ≤ 毛利 ＜ MaxAmount 的阶梯，按 LadderSeq 排序取第一条（通常设计为不重叠区间）。
  • 若未找到匹配阶梯，则匹配提成比例为 NULL，后续将使用最高比例作为默认。

📤 输出字段
所有毛利维度字段 + Ladder_Id, SchemeName, 最高提成比例, 匹配提成比例。

💡 注意：负毛利单独处理，直接使用最高比例，体现激励规则。

步骤 6：最终输出（SELECT）

🎯 业务目标
生成最终提成报表，包含提成比例（带状态标注）和提成金额，并增加固定字段（费用类型、业务类型）和预估年月。

📥 输入数据
来自 提成计算 CTE。

⚙️ 输出字段计算

  • 提成比例（显示用）：
    • 若未匹配方案 → '未设置提成规则'
    • 若毛利 < 0 → 显示 最高提成比例% (负数毛利)
    • 否则 → 显示 匹配提成比例%，若匹配为 NULL 则显示 最高提成比例% (默认)
  • 提成金额：
    • 未匹配方案 → 0
    • 毛利 < 0 → 毛利 * 最高提成比例
    • 其他 → 毛利 * (匹配提成比例 或 最高提成比例)
  • 预估年月：当前系统日期（GETDATE()）的 yyyy-MM 格式。
  • 费用类型：固定为“销售费用”。
  • 业务类型：固定为空字符串（保留字段）。

📤 最终输出
一张完整的提成明细表，可直接用于报表展示或后续核算。

五、业务规则说明

  • 代拉代报转换规则：
    若原始业务大类为「代拉代报」且为分单（是否分单 = 1）且主单业务大类不为空，则转换为「主单业务大类」； 否则保持「代拉代报」。
  • 跨境电商合并规则：
    「跨境电商头程」「跨境电商尾程」「跨境电商全段」统一合并为「跨境电商」。
  • 提成比例匹配规则：
    正毛利按 ABS(毛利金额) 匹配阶梯区间（MinAmount ≤ 毛利 ＜ MaxAmount）； 负毛利直接使用该阶梯的 最高提成比例。
  • 未匹配方案处理：
    若未匹配到任何提成方案（Ladder_Id IS NULL），则提成金额为 0，提成比例显示「未设置提成规则」。
  • 数据过滤：
    过滤已取消（Cancelid = 'T'）、红冲（redid IS NOT NULL）、业务员为空、业务大类为空或「海外仓」的记录。
  • 日期参数：
    使用 $DE_PARAM{...} 动态传入 BEGINTIME 和 ENDTIME，支持参数化查询。

-- ============================================================ -- 销售提成计算 SQL（完整版） -- 数据库：SQL Server -- ============================================================ -- 1. 按 CostId 聚合毛利，保留原始业务大类及分单信息 WITH 成本毛利 AS ( SELECT c.Id AS CostId, RTRIM(LTRIM(c.ctrloffice)) AS 接单公司, c.Saler AS 业务员, FORMAT(c.BizDate, 'yyyy-MM') AS 月度, COALESCE(e_main.BusWay, c.BusWay) AS 原始业务大类, MAX(CASE WHEN e_main.SplitId = 'P' THEN e_main.BusWay END) AS 主单业务大类, MAX(CASE WHEN e_self.SplitMblId IS NOT NULL THEN 1 ELSE 0 END) AS 是否分单, ROUND( SUM(CASE WHEN ci.Ioid = 1 THEN CASE WHEN ci.ChgSign = '*' THEN ci.Amount * ISNULL(ci.ChgRate, 1) WHEN ci.ChgSign = '/' THEN ci.Amount / ISNULL(ci.ChgRate, 1) ELSE ci.Amount END ELSE 0 END) - SUM(CASE WHEN ci.Ioid = -1 THEN CASE WHEN ci.ChgSign = '*' THEN ci.Amount * ISNULL(ci.ChgRate, 1) WHEN ci.ChgSign = '/' THEN ci.Amount / ISNULL(ci.ChgRate, 1) ELSE ci.Amount END ELSE 0 END), 2) AS 毛利金额 FROM Cost c INNER JOIN CostItem ci ON c.Id = ci.CostId LEFT JOIN Export e_self ON c.Id = e_self.Id LEFT JOIN Export e_main ON e_self.SplitMblId = e_main.SplitMblId AND e_main.SplitId = 'P' WHERE 1=1 AND $DE_PARAM{ c.BizDate >= '$[BEGINTIME]' AND c.BizDate < DATEADD(DAY, 1, '$[ENDTIME]') } AND c.Cancelid <> 'T' AND c.Saler IS NOT NULL AND c.Saler <> '' AND ci.redid IS NULL AND COALESCE(e_main.BusWay, c.BusWay) IS NOT NULL AND RTRIM(LTRIM(COALESCE(e_main.BusWay, c.BusWay))) <> '' AND RTRIM(LTRIM(COALESCE(e_main.BusWay, c.BusWay))) <> '海外仓' GROUP BY c.Id, c.ctrloffice, c.Saler, c.BizDate, COALESCE(e_main.BusWay, c.BusWay) ), -- 2. 业务大类转换 毛利数据 AS ( SELECT m.接单公司, RTRIM(LTRIM( CASE WHEN m.原始业务大类 = '代拉代报' THEN CASE WHEN m.是否分单 = 1 AND m.主单业务大类 IS NOT NULL THEN m.主单业务大类 ELSE '代拉代报' END WHEN m.原始业务大类 IN ('跨境电商头程', '跨境电商尾程', '跨境电商全段') THEN '跨境电商' ELSE m.原始业务大类 END )) AS 实际业务大类, m.业务员, m.月度, SUM(m.毛利金额) AS 毛利金额 FROM 成本毛利 m WHERE m.原始业务大类 IS NOT NULL AND RTRIM(LTRIM(m.原始业务大类)) <> '海外仓' GROUP BY m.接单公司, CASE WHEN m.原始业务大类 = '代拉代报' THEN CASE WHEN m.是否分单 = 1 AND m.主单业务大类 IS NOT NULL THEN m.主单业务大类 ELSE '代拉代报' END WHEN m.原始业务大类 IN ('跨境电商头程', '跨境电商尾程', '跨境电商全段') THEN '跨境电商' ELSE m.原始业务大类 END, m.业务员, m.月度 ), -- 3. 拆分提成方案（只取业务提成） 方案拆分 AS ( SELECT s.Ladder_Id, s.SchemeName, s.BusinessType, LTRIM(RTRIM(Split.a.value('.', 'NVARCHAR(MAX)'))) AS 接单公司 FROM Zj_CommissionScheme s CROSS APPLY (SELECT CAST('<M>' + REPLACE(s.CtrlOffice, ',', '</M><M>') + '</M>' AS XML) AS x) t CROSS APPLY t.x.nodes('/M') Split(a) WHERE s.SchemeType = 'SALER' AND s.CalcType = 'SINGLE' AND s.CtrlOffice IS NOT NULL AND s.CtrlOffice != '' AND s.Status = 1 AND (s.EffectiveDate <= GETDATE() OR s.EffectiveDate IS NULL) AND (s.ExpirationDate >= GETDATE() OR s.ExpirationDate IS NULL) ), -- 4. 方案匹配（去重） 方案匹配 AS ( SELECT DISTINCT s.接单公司, s.BusinessType, s.Ladder_Id, s.SchemeName FROM 方案拆分 s ), -- 5. 提成计算 提成计算 AS ( SELECT m.接单公司, m.实际业务大类, m.业务员, m.月度, m.毛利金额, sm.Ladder_Id, sm.SchemeName, (SELECT MAX(CommissionRate) FROM Zj_CommissionLadder l WHERE l.Ladder_Id = sm.Ladder_Id) AS 最高提成比例, (SELECT TOP 1 l.CommissionRate FROM Zj_CommissionLadder l WHERE l.Ladder_Id = sm.Ladder_Id AND (l.MinAmount IS NULL OR ABS(m.毛利金额) >= l.MinAmount) AND (l.MaxAmount IS NULL OR ABS(m.毛利金额) < l.MaxAmount) ORDER BY l.LadderSeq) AS 匹配提成比例 FROM 毛利数据 m LEFT JOIN 方案匹配 sm ON m.接单公司 = sm.接单公司 AND m.实际业务大类 = sm.BusinessType ) -- 6. 最终输出 SELECT t.接单公司, t.实际业务大类 AS 业务大类, t.业务员, t.月度, FORMAT(GETDATE(), 'yyyy-MM') AS 预估年月, t.毛利金额, '销售费用' AS 费用类型, '' AS 业务类型, CASE WHEN t.Ladder_Id IS NULL THEN '未设置提成规则' WHEN t.毛利金额 < 0 THEN CONVERT(VARCHAR(50), ROUND(ISNULL(t.最高提成比例, 0) * 100, 2)) + '% (负数毛利)' ELSE ISNULL( CONVERT(VARCHAR(50), ROUND(t.匹配提成比例 * 100, 2)) + '%', CONVERT(VARCHAR(50), ROUND(ISNULL(t.最高提成比例, 0) * 100, 2)) + '% (默认)' ) END AS 提成比例, CASE WHEN t.Ladder_Id IS NULL THEN 0 WHEN t.毛利金额 < 0 THEN ROUND(t.毛利金额 * ISNULL(t.最高提成比例, 0), 2) ELSE ROUND(t.毛利金额 * ISNULL(t.匹配提成比例, ISNULL(t.最高提成比例, 0)), 2) END AS 提成金额 FROM 提成计算 t ORDER BY t.接单公司, t.实际业务大类, t.业务员, t.月度;
